## Tech Stack

- **Next.js**: framework React para SSR/SSG, rotas e otimizações nativas.
- **TypeScript**: tipagem estática para maior segurança e manutenção.
- **Tailwind CSS**: estilização utilitária com consistência visual e produtividade.
- **shadcn/ui**: componentes reutilizáveis e acessíveis, com bom baseline de UX.
- **Framer Motion**: animações fluidas e microinterações.
- **Lucide**: ícones consistentes e leves.

## Por que essas escolhas?

- **Next.js**: melhora **SEO** e **performance** com SSR/SSG, otimização de assets e uma base sólida para evoluir o site sem perder velocidade.
- **TypeScript**: reduz bugs em refactors, deixa contratos claros e melhora a **manutenibilidade** (especialmente em componentes e UI).
- **Tailwind CSS**: mantém consistência visual com baixo overhead, facilita iterar no design e ajuda a evitar CSS “crescendo sem controle”.
- **shadcn/ui**: acelera a construção de UI com componentes reutilizáveis e padrões que favorecem **acessibilidade** e **boa UX**, sem “lock-in” de biblioteca fechada.
- **Framer Motion**: entrega microinterações com qualidade sem complexidade excessiva, elevando a percepção de produto sem sacrificar a base do layout.
- **Lucide**: ícones leves e consistentes, com ótima integração com React e impacto mínimo no bundle.

## Objetivos do projeto

- **SEO**: páginas prontas para indexação, metadados bem definidos e boas práticas para visibilidade orgânica.
- **Acessibilidade (a11y)**: componentes e padrões com foco em navegação por teclado, semântica e contraste.
- **Performance**: otimizações para carregamento rápido e boa experiência (Core Web Vitals).

## Como rodar localmente

### Pré-requisitos

- **Node.js** (recomendado: versão LTS)
- **npm** (este repositório utiliza `package-lock.json`)

### Instalação e execução

```bash
# 1) Instalar dependências
npm install

# 2) Rodar em desenvolvimento
npm run dev
```

Acesse `http://localhost:3000`.

### Scripts úteis

```bash
# build de produção
npm run build

# iniciar servidor de produção (após build)
npm run start

# lint
npm run lint
```
